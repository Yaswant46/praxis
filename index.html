How It Works

Four roles.
One company.
Infinite pressure.

01
Game Master Sets the Stage
The facilitator creates a session, configures teams with access codes, and selects the simulation case. Participants get a shareable link.
02
Teams Enter the Company
Each team is a leadership group inside the same company. They read the case, study the financials, and face five decision domains every quarter.
03
Curveballs Land Mid-Round
The Game Master injects live scenario events — competitor launches, policy shifts, supply chain shocks — that hit all teams simultaneously. No warning.
04
Outcomes Are Published
After each quarter, the GM scores team decisions and publishes metrics. Dashboards update live. The leaderboard reflects compounding consequences.
05
Structured Debrief
Each quarter closes with an Assessment page — not scores alone, but calibrated strategic questions designed to surface the reasoning behind each decision.

Platform

Built for facilitators
who take outcomes seriously.

Everything runs in the browser. No installs. No complex setup. One link gets a cohort of 60 people into a live simulation.

⚡
Live Curveball Engine
Game Masters inject scenario events in real time. Every team sees the curveball instantly on their screen — no refresh, no lag, broadcast to all simultaneously.
📊
Live Team Dashboards
Each team has a live dashboard showing KPIs, metric bars, round progress, and a scoring leaderboard. Updates push automatically as the GM publishes outcomes.
🔒
Email-Locked Team Slots
Each team slot is locked to a single email on first login. Prevents unauthorised access mid-simulation. First person with the correct code claims and locks the slot.
◈
Five Decision Domains
Engineering, Manufacturing, Supply Chain, Commercial, and People — every quarter. Each domain has structured decision fields designed to surface trade-off reasoning.
◎
Calibrated Assessments
Every completed quarter generates a structured assessment for each team — not just scores, but five strategic questions designed to drive debrief conversations.
💾
Draft Autosave
Decision forms autosave locally as teams fill them in. If a tab closes or a browser crashes mid-fill, the draft restores on next load. No lost work.
▲
Live Leaderboard
The leaderboard shows team scores, metric trajectories, and round-on-round deltas. Updates automatically when the GM publishes outcomes after each quarter.
📋
Decision Log
Every submitted decision is stored and visible to each team in a structured history view, quarter by quarter. The full reasoning trail is preserved for debrief.
🛡
OTP-Verified Game Masters
Game Masters verify via one-time email code before accessing the admin panel. No passwords to manage. The Master Admin can create and monitor all sessions.
Roles

Three ways to
be inside Praxis.

🔑
Master Admin
The programme operator. Creates and oversees all sessions across cohorts. Full visibility into every session.
  • Create sessions for any case and assign Game Masters
  • Configure team names, emails, and access codes
  • Monitor all sessions from a single dashboard
  • Enter any session as Game Master when needed
  • Inject curveballs into any active session
🛡
Game Master
The facilitator. Runs a single session in real time — managing rounds, triggering events, and publishing outcomes.
  • Open and close decision windows each quarter
  • Inject curveball events broadcast to all teams
  • Review team submissions and enter outcome scores
  • Advance the simulation quarter by quarter
  • End the session and trigger the debrief state
◈
Participant
The decision-maker. A member of a leadership team inside the simulated company, making calls with incomplete information.
  • Read the full case brief with financials and challenges
  • Make structured decisions across five domains each quarter
  • Receive curveball events and live metric updates
  • View assessments, scores, and the leaderboard
  • Retain a full decision log for the debrief session
